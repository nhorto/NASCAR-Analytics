# Core Beliefs

> Opinionated decisions that guide how this product is built. These are non-negotiable.

1. **Proprietary metrics over raw data** — Anyone can show NASCAR's loop data. Our value is the unique stats we compute on top of it. If a metric exists elsewhere, we need to do it better or create something new.

2. **Mobile-first, not mobile-friendly** — Design for the phone screen first. The person checking stats at the track or on the couch is our primary user. Desktop is a bonus.

3. **Freemium is the growth engine** — The free tier must be genuinely useful (not a teaser). It drives SEO, word-of-mouth, and trust. Premium unlocks predictions, optimizers, and advanced tools.

4. **Race-weekend rhythm** — The product peaks on race weekends. Pre-race analysis (Thursday-Saturday) and post-race data (Sunday-Monday) are the two content beats. Build for this cadence.

5. **Speed is a feature** — If the site takes more than 1 second to load data, it's broken. Pre-compute and cache aggressively. Users are comparing us to apps, not websites.

6. **Don't fight NASCAR's data model** — Use their loop data terminology and metrics as the foundation. Fans already know "Driver Rating" and "Quality Passes." Build on that vocabulary, don't replace it.

7. **The database is the product** — 20 years of historical loop data, normalized and queryable, is the real asset. The UI is just a window into it. Invest in data quality above all else.
